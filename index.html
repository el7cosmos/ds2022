From NGINX + php-fpm to NGINX Unit

Containerization

VS
When talking about containerization in PHP using NGINX, I've seen the community is divided into two arguments, should NGINX and php-fpm run in two separate containers, or should they run together in a single container? Running NGINX and php-fpm in separate containers requires the PHP application to reside in a shared volume that is mounted to both containers. The other side of the argument is to run both NGINX and php-fpm in a single container. This usually comes with the php application residing inside the container without mounted volume and copied when building the image. If we can't use two separate containers, or can't mount the application to the container, for example in Platform as a Service like (a) or (b), or some organization policy, we have to choose this option.

Containerization

VS
Now with NGINX Unit, we don't have to worry about that anymore, because it is only one service, so one container is enough, we can mount the application to the container, or we can copy the application to the container when building the image. So it fits all our needs, whichever it is.
